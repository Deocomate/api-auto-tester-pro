API Configuration Guide

To use the API Auto Tester, you need to provide a JSON file containing an array of API endpoint configurations. Each object in the array represents a single API request to be tested.

JSON File Structure:

The JSON file must be an array of objects. Each object can have the following fields:

  • endpoint (string, required): The full URL of the API endpoint.
  • method (string, required): The HTTP method (e.g., "GET", "POST", "PUT", "DELETE").
  • data (object, optional): The request payload for methods like POST, PUT, PATCH.
  • headers (object, optional): Custom headers for the request (e.g., {"X-Custom-Header": "value"}). The Bearer token will be added automatically if provided in the Authentication section.
  • params (object, optional): URL query parameters (e.g., {"id": 1, "sort": "asc"}).
